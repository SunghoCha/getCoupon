# getCoupon 프로젝트 개요

## 서비스 컨셉

겟쿠폰은 커뮤니티 활동을 통해 포인트를 얻고, 광고주(쿠폰발급자)는 자신들의 쿠폰을 올려 상품을 홍보하는 플랫폼이다.
유저는 포인트 또는 추첨, 선착순 등의 방식으로 쿠폰을 얻을 수 있다.

## 핵심 가치

쿠폰 발급 전략의 다양성이 이 프로젝트의 기술적 핵심이다.
발급 전략은 자주 변하는 핵심 비즈니스이며, 아키텍처 설계의 중심이 된다.

## 프로젝트 범위

### 핵심 도메인 (집중)

- **쿠폰 발급 전략** — 선착순, 포인트 구매, 포인트 추첨, 경매, 일반 추첨 등
- **쿠폰 라이프사이클** — 생성 → 검수 → 공개 → 응모 → 발급 → 사용/만료

### 지원 도메인 (최소 구현)

- **회원** — 가입 즉시 활동 가능. 단순하게
- **쿠폰발급자** — 회원 중 자격 조건 충족 + 승인 과정을 거쳐 전환
- **포인트** — 적립/차감. 쿠폰 발급 전략과 연동되는 핵심 재화
- **커뮤니티** — 글/댓글 정도만. 포인트 적립 이벤트 트리거 역할

### 액터

- **회원** — 커뮤니티 활동, 포인트 적립, 쿠폰 응모/사용
- **쿠폰발급자** — 쿠폰 생성, 검수 요청
- **관리자** — 발급자 승인, 쿠폰 검수 승인

## 아키텍처 방향

- **헥사고날 아키텍처**
- **그래들 멀티모듈**
- **DDD 차용** (완전한 DDD는 아니지만 방식을 많이 활용)
- 이벤트스토밍 → 도메인 모델 설계 → TDD 순서로 진행

## 기술 스택

- Java 21, Spring Boot 4.0.5
- MySQL, Redis, Kafka (필요한 경우)
- Testcontainers

## 테스트 전략

우선순위: 도메인 모델 단위 테스트 → 애플리케이션(유스케이스) 테스트 → 통합 테스트
컨트롤러 슬라이스 테스트(MockMvc)보다 요청이 DB까지 관통하는 통합 테스트를 선호한다.

## 포트폴리오 목적

백엔드 취업용 포트폴리오. 쿠폰 발급 전략의 아키텍처적 고민과 기술적 깊이에서 임팩트를 보여준다.
