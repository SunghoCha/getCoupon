# letterPick 프로젝트 개요

## 서비스 컨셉

letterPick은 뉴스레터 종합 리딩 플랫폼이다.
흩어진 뉴스레터를 한곳에 모아 읽고, 텍스트 분석을 통해 단순 모아보기 이상의 가치를 제공한다.

## 핵심 가치

기존 뉴스레터 플랫폼은 사용자가 구독한 레터만 모아서 보여주는 수준에 그친다.
letterPick은 서버에 축적된 모든 발행처의 콘텐츠를 텍스트 분석하여,
사용자가 구독하지 않은 레터에서도 관심 있는 정보를 **발견**할 수 있도록 한다.

### 차별점

| | 기존 플랫폼 (모아보기) | letterPick (발견) |
|---|---|---|
| 콘텐츠 범위 | 내가 구독한 것만 | 모든 발행처의 레터 |
| 사용자 행동 | 도착한 걸 읽기 (수동적) | 트렌드 태그로 탐색 → 발견 → 구독 (능동적) |
| 상단 영역 | 광고/장식 | 실시간 트렌드 태그 (클릭 → 관련 레터 조회) |
| 놓치는 정보 | 구독 안 하면 모름 | 태그/추천으로 구독 외 레터 노출 |

## 비즈니스 흐름

### 구독 흐름
1. letterPick에 발행처 목록이 미리 등록되어 있음
2. 사용자가 목록에서 구독하고 싶은 발행처 클릭 → 전용 이메일 주소 복사
3. 사용자가 외부 뉴스레터 사이트에서 해당 이메일로 구독 등록
4. 외부 서비스가 웰컴 메일/최신호 발송 → letterPick 서버에 도착
5. 첫 메일 도착 시 구독 관계 자동 생성 + 콘텐츠 저장

### 콘텐츠 저장 구조
- 메일은 사용자별로 수신되지만, 콘텐츠는 **발행처 단위로 중복 제거하여 저장**
- 같은 뉴스레터를 100명이 구독해도 콘텐츠 저장은 1번, 구독 관계만 100개
- 발행처별로 콘텐츠를 보유하고 있으므로 구독하지 않은 사용자에게도 노출 가능

### 사용자 뷰
- **투데이** — 오늘 도착한 레터만 필터링
- **보관함** — 구독 중인 전체 레터 목록
- **트렌드 태그** — 전체 레터에서 추출한 핵심 키워드를 상단에 표시, 클릭 시 관련 레터 조회 (구독 외 발행처 포함)

## 시스템 큰 줄기

### 줄기 1: 메일 수신 파이프라인
- 사용자별 전용 이메일 주소 발급 (예: user123@letterpick.com)
- 수신자 주소에서 사용자 식별 (라우팅 키 역할)
- 메일 수신 → 파싱 → 발행처 단위 저장 (중복 제거)

### 줄기 2: 검색 + Elasticsearch 활용
- 검색 발전: LIKE → Full-Text Search → Elasticsearch + Nori
- Elasticsearch 도입 후 확장:
  - **태그 자동 추출** — Nori Aggregation으로 레터별 핵심 키워드 추출
  - **트렌드 태그** — 전체 레터 기준 최근 키워드 빈도 집계, 화면 상단에 표시
  - **연관 레터** — More Like This 쿼리로 키워드가 겹치는 레터 추천
  - **개인화 추천** — 사용자가 읽은 레터들의 키워드를 기반으로 관심사에 맞는 레터 추천

## 바운디드 컨텍스트

### 회원 컨텍스트
- 가입, 전용 이메일 주소 발급
- 독립적인 컨텍스트

### 뉴스레터 컨텍스트
- **발행처 관리** — 발행처 등록/조회 (관리자)
- **메일 수신** — 외부 메일 수신, 파싱 (인바운드 어댑터)
- **레터 저장** — 발행처 단위 중복 제거하여 저장
- **구독** — 첫 메일 도착 시 자동 생성, 구독 목록 관리, 해지
- **읽음 상태** — 사용자별 읽음 처리
- **초기 검색** — LIKE / Full-Text Search (Elasticsearch 도입 전)

### 검색/분석 컨텍스트
- Elasticsearch 도입 시점에 뉴스레터 컨텍스트에서 분리
- **검색** — Elasticsearch + Nori 기반 한글 검색, 자동완성, 오타 보정
- **태그 추출** — Nori Aggregation으로 레터별 핵심 키워드 추출
- **트렌드 태그** — 전체 레터 기준 키워드 빈도 집계
- **연관 레터** — More Like This 쿼리로 유사 레터 추천
- **개인화 추천** — 사용자가 읽은 레터의 키워드 기반 추천

### 후순위 (별도 프로젝트로 분리 가능)
- **쿠폰/이벤트** — 구독 쿠폰, 선착순 이벤트 (동시성 제어, Redis 분산 락)

### 액터
- **회원** — 뉴스레터 구독, 열람, 검색, 트렌드 태그 탐색
- **관리자** — 뉴스레터 발행처 관리, 시스템 운영

## 아키텍처 방향

- **헥사고날 아키텍처**
- **그래들 멀티모듈**
- **DDD 차용** (완전한 DDD는 아니지만 방식을 많이 활용)
- 이벤트스토밍 → 도메인 모델 설계 → TDD 순서로 진행

## 기술 스택

- Java 21, Spring Boot 4.0.5
- MySQL
- Elasticsearch + Nori (검색, 형태소 분석, 태그 추출, 트렌드 집계)
- Testcontainers
- Redis (후순위: 선착순 이벤트 동시성 제어)

## 테스트 전략

우선순위: 도메인 모델 단위 테스트 → 애플리케이션(유스케이스) 테스트 → 통합 테스트
컨트롤러 슬라이스 테스트(MockMvc)보다 요청이 DB까지 관통하는 통합 테스트를 선호한다.

## 포트폴리오 목적

백엔드 취업용 포트폴리오.
메일 수신 파이프라인의 비동기 처리 설계와 텍스트 분석/검색 시스템의 기술적 깊이에서 임팩트를 보여준다.
