# getCoupon 아키텍처 가이드

## 아키텍처

- 헥사고날 아키텍처
- 도메인 모델 패턴
- 그래들 멀티모듈 (모듈 구성은 이벤트스토밍 이후 결정)

## 계층

```
외부(Actor) → 어댑터 → 애플리케이션 → 도메인
```

### Domain 계층
- 순수 Java. 다른 계층에 의존하지 않음
- 애그리거트, 엔티티, Value Object, 도메인 서비스 인터페이스, 도메인 이벤트
- 비즈니스 규칙과 불변식이 여기에 위치

### Application 계층
- 도메인 계층에만 의존
- 유스케이스 구현 (트랜잭션 경계)
- provided(인바운드 포트)와 required(아웃바운드 포트)로 분리

### Adapter 계층
- 애플리케이션 계층의 provided를 호출하고, required를 구현
- webapi(REST), persistence(JPA), integration(외부 연동), security 등

## 의존성 방향

```
adapter → application → domain
```

ArchUnit 테스트로 의존 방향을 강제한다.

```java
Architectures.layeredArchitecture()
    .layer("domain").definedBy("..domain..")
    .layer("application").definedBy("..application..")
    .layer("adapter").definedBy("..adapter..")
    .whereLayer("domain").mayOnlyBeAccessedByLayers("application", "adapter")
    .whereLayer("application").mayOnlyBeAccessedByLayers("adapter")
    .whereLayer("adapter").mayNotBeAccessedByAnyLayer()
```

## 패키지 구조

```
com.sungho.getcoupon/
├── domain/
│   ├── shared/              # 공유 Value Object
│   └── {aggregate}/         # 애그리거트별 패키지
│       ├── Entity.java
│       ├── Status.java      # 상태 enum
│       ├── ValueObject.java
│       └── DomainService.java  # 인터페이스
│
├── application/
│   └── {aggregate}/
│       ├── XxxService.java     # 유스케이스 구현
│       ├── provided/           # 인바운드 포트 (외부에 제공하는 인터페이스)
│       └── required/           # 아웃바운드 포트 (외부에 요구하는 인터페이스)
│
└── adapter/
    ├── webapi/                 # 인바운드 어댑터 (REST API)
    │   └── dto/
    ├── persistence/            # 아웃바운드 어댑터 (JPA 구현 등)
    ├── integration/            # 아웃바운드 어댑터 (외부 시스템 연동)
    └── security/               # 아웃바운드 어댑터
```

## 멀티모듈 구성

> 이벤트스토밍 이후 바운디드 컨텍스트가 도출되면 결정한다.
> 모듈 경계 = 아키텍처 경계가 되도록 설계할 예정.

## 코딩 스타일

### 도메인 엔티티
- 생성은 `static` 팩토리 메서드 사용
- 상태 전이는 의미 있는 메서드로 표현 (setter 사용 금지)
- 불변식은 `Assert.state()`로 선언적 표현
- null 검증은 `requireNonNull` 사용
- `@Getter` 사용, setter 없음
- `@NoArgsConstructor(access = PROTECTED)` — JPA용

### 애플리케이션 서비스
- 조회(Query)와 변경(Modify)을 별도 서비스로 분리
- provided 인터페이스를 구현
- 도메인 불변식 중 외부 조회가 필요한 것(중복 체크 등)은 서비스에서 검증

### 어댑터
- Controller는 provided 포트만 의존
- 도메인 객체를 직접 반환하지 않고 DTO로 변환

### 패키지 레벨
- `@NonNullApi` — domain, application 패키지에 적용

## 테스트 전략

### 우선순위
1. **도메인 모델 단위 테스트** — 순수 Java, 인프라 의존 없이 수십 ms 수준
2. **애플리케이션 유스케이스 테스트** — 포트 기반 테스트
3. **통합 테스트** — 컨트롤러 → DB까지 관통. Testcontainers 활용

### 아키텍처 테스트
- ArchUnit으로 계층 간 의존 방향 강제

## 기술 스택

- Spring Web MVC, JPA, Validation
- Lombok
- MySQL, Redis, Kafka (필요시)
